# /develop 워크플로우

새 기능을 개발하는 전체 프로세스입니다.

## 워크플로우 단계

### 1. 요구사항 분석 (Orchestrator)

사용자 요청을 분석하고 구현 범위를 정의합니다.

```
- 요청 내용 명확화
- 기능 요구사항 도출
- 비기능 요구사항 확인 (성능, 보안 등)
- 구현 범위 결정
```

### 2. 기존 코드 조사 (Researcher)

기존 코드베이스를 분석하여 구현에 필요한 정보를 수집합니다.

```
- 관련 파일/모듈 탐색
- 기존 패턴 파악
- 재사용 가능한 코드 식별
- 의존성 확인
```

// turbo
### 3. 구현 계획 수립 (Orchestrator)

분석 결과를 바탕으로 구현 계획을 작성합니다.

```
- 변경/생성할 파일 목록
- 구현 순서 결정
- 예상 이슈 파악
- 계획 사용자 확인 요청
```

### 4. 코드 작성 (Coder)

계획에 따라 코드를 구현합니다.

```
- 새 파일 생성 또는 기존 파일 수정
- 테스트 코드 작성
- 코드 품질 가이드라인 준수
- 적절한 주석 추가
```

// turbo
### 5. 테스트 실행

구현된 코드가 정상 동작하는지 확인합니다.

```bash
# 프로젝트에 맞는 테스트 명령 실행
npm test
# 또는
npm run dev  # 개발 서버로 수동 확인
```

### 6. Handoff 생성 (Coder)

코드 검토를 위해 HANDOFF.md를 작성합니다.

```markdown
templates/HANDOFF.md 템플릿 사용:
- 변경 파일 목록
- 구현 내용
- 설계 결정 사항
- 테스트 방법
```

### 7. 코드 검토 (Reviewer - 새 세션)

> ⚠️ **중요**: 새 대화 세션에서 `/review` 워크플로우를 실행하세요.
> Context Bias를 제거하기 위해 같은 세션에서 검토하지 않습니다.

```
새 세션에서:
/review
```

### 8. 문서화 (Documenter)

필요한 문서를 작성하거나 업데이트합니다.

```
- README 업데이트 (필요시)
- API 문서 작성 (API 변경시)
- 사용자 가이드 업데이트
```

## 완료 체크리스트

- [ ] 요구사항이 모두 구현되었는가?
- [ ] 테스트가 통과했는가?
- [ ] HANDOFF.md가 작성되었는가?
- [ ] 코드 리뷰가 완료되었는가?
- [ ] 문서가 업데이트되었는가?

## 예시

**요청**: `/develop 사용자 프로필 수정 기능 추가`

**실행 흐름**:
1. 프로필 수정 기능 요구사항 분석
2. 기존 사용자 관련 코드 조사
3. PUT /api/users/:id 엔드포인트 계획
4. 코드 구현 및 테스트
5. HANDOFF.md 작성
6. (새 세션) /review 실행
7. README 또는 API 문서 업데이트
